Primeiros comandos Git

ls -> lista quais itens tem no diretório
cd -> Muda de diretório
pwd -> Aonde estou? Mostra o caminho até o diretório que estou
mkdir -> Criar uma nova pasta
cd .. -> sobe uma pasta anterior
cd . -> é a própria pasta
cat -> mostra o que tem dentro do arquivo
git init -> torna a (pasta especial). A pasta vira um repositório git
git status -> mostra como está meu diretório
git add -> me ajudar a colocar o arquivo que vai para o commit
git commit -m "descrição do commit" -> criar um commit e colocar descrição
git config user.name -> verificar meu nome da minha conta git
git config user.email
nano "nome arquivo que a gente vai criar" -> editor de texto dentro do bash, pode editar e criar um arquivo
ctrl + o - salva o arquivo + enter
ctrl + x - para sair

git log -> mostra todos o histórico de commit

## FLUXO DE TRABALHO

1. criar o diretório (pasta) do projeto
2. iniciar o git dentro do diretório criado(git init .)
3. edita ou cria arquivos
4. git status
5. git add <nome arquivo>
7. git commit -m "mensagem"
8. git status
9. volta para o passo 3

git diff "nome arquivo" -> mostra o que foi alterado no arquivo
git diff . -> mostra todas as alterações dentro do diretório
git reset "nome arquivo" -> tira o arquivo do commit(deixa de ficar verde para vermelho)

git checkout -b "nome branch" -> cria uma cópia dos arquivos, não é o mesmo ramo(ramificação)
